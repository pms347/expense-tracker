Step 1 — Create Firebase Project
1
Go to console.firebase.google.comClick "Add project" → give it a name → Continue → Create project.
2
Enable AuthenticationBuild → Authentication → Get started → Email/Password → Enable → Save. Then Providers → Google → Enable → Save.
3
Enable FirestoreBuild → Firestore Database → Create database → Production mode → Choose region → Done.
Step 2 — Firestore Security Rules
In Firestore → Rules tab, replace everything with:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{userId}/{document=**} { allow read, write: if request.auth != null && request.auth.uid == userId; } } }
Step 3 — Get Config & Paste
4
Project Settings → General → Your apps → Add web appCopy the firebaseConfig object. Paste it into the CONFIG section at the top of index.html.
const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "your-app.firebaseapp.com", projectId: "your-app-id", storageBucket: "your-app.appspot.com", messagingSenderId: "123456789", appId: "1:123:web:abc123" };
Step 4 — Deploy to GitHub Pages
5
Add your domain to Firebase AuthAuthentication → Settings → Authorized domains → Add your GitHub Pages domain (e.g. username.github.io).
6
Upload to GitHub & enable PagesSettings → Pages → main branch → Save. Live at username.github.io/expense-tracker/
Common Issues
⚠ Google Sign-In popup blocked
Make sure your domain is added in Firebase → Authentication → Settings → Authorized domains. On mobile, popups may be blocked — the app falls back to redirect automatically.
⚠ "Permission denied" on Firestore
Your Security Rules may be incorrect. Go to Firestore → Rules and paste the rules from the Setup tab above.
⚠ Data not loading on another device
Make sure you're signed in with the same account. Tap the sync icon or reopen the app — Firestore loads fresh data on every login.
⚠ Budget overview empty
Go to 🏷 Categories → enter a ₹ amount in the budget field next to any category.
⚠ App not installable as PWA
Make sure manifest.json is in the repo and the link tag is in <head>. GitHub Pages uses HTTPS which is required for PWA install.
⚠ authDomain error after deploy
Add your GitHub Pages URL to Firebase Console → Authentication → Settings → Authorized domains.
