Tech Stack
🌐
HTML5 + CSS3 + JavaScriptThe entire app is a single index.html file — no build tools, no npm required.
🗄
Supabase (PostgreSQL)Free cloud database with built-in authentication and Row Level Security for per-user data isolation.
📊
Chart.js v4.4Powers the donut (category) and bar (trend) charts.
📄
jsPDF + AutoTableGenerates downloadable PDF expense reports with dark styling.
📱
PWA (Progressive Web App)Installable on Android via Chrome. Opens full-screen, no browser bar. Works on Nothing Phone.
Architecture

The app uses a Cache-First, Cloud-Update strategy. On open it instantly shows data from localStorage, then silently fetches fresh data from Supabase and updates the screen. Every user's data is isolated by their user_id — nobody can see anyone else's expenses.

Core Features
🔐
User AuthenticationSign up and sign in with email + password. Each user's data is completely private. Powered by Supabase Auth.
☁️
Cloud SyncAll data lives in Supabase. Opens on any browser, any device, incognito — always your data.
💾
Offline CacheData is also saved in localStorage so the app loads instantly without waiting for the server.

✏️
Edit & DeleteTap the ✏ button on any expense to edit it. Changes sync to cloud immediately.

Step 1 — Create Supabase Project
1
Go to supabase.comSign up free (Google login works). Click "New Project" → give it a name → set a password → Create. Wait ~2 minutes.
2
Open SQL EditorIn your project, click "SQL Editor" in the left sidebar.
3
Run this SQL (copy and paste all of it)
create table if not exists expenses ( id bigint primary key generated always as identity, user_id uuid references auth.users(id) on delete cascade, name text not null, amt numeric not null, cat text not null, date text not null, note text default '', created_at timestamptz default now() ); create table if not exists categories ( id bigint primary key generated always as identity, user_id uuid references auth.users(id) on delete cascade, name text not null, color text not null, budget numeric default 0 ); alter table expenses enable row level security; alter table categories enable row level security; create policy "own_exp" on expenses for all to authenticated using (auth.uid()=user_id) with check (auth.uid()=user_id); create policy "own_cats" on categories for all to authenticated using (auth.uid()=user_id) with check (auth.uid()=user_id);
Step 2 — Get Your Credentials
4
Go to Project Settings → APICopy your Project URL (https://xxxx.supabase.co) and anon public key (starts with eyJ...).
5
Paste into index.htmlFind the CONFIG section at the top of the script tag and fill in both values.
const SB_URL = 'https://xxxx.supabase.co'; const SB_KEY = 'eyJhbGciOiJIUzI1NiIs...';
Step 3 — Deploy to GitHub Pages
6
Create a GitHub repogithub.com → New Repository → "expense-tracker" → Public.
7
Upload index.html + manifest.jsonAdd file → Upload files → Commit.
8
Enable PagesSettings → Pages → Source: main branch → Save. Your URL: yourusername.github.io/expense-tracker/

Common Issues
⚠ Login button stays disabled after logout
The resetAuthUI() function should re-enable it. If stuck, refresh the page. This is fixed in v2.0 with onAuthStateChange listener.
⚠ Data not loading on another browser
Make sure you ran the SQL with RLS policies in Supabase. Also ensure you are logged in with the same account.
⚠ "↻ Syncing" badge stays forever or shows Error
Check SB_URL and SB_KEY are correct. Verify RLS SQL was run. Open Supabase → Table Editor and confirm the tables exist.
⚠ Budget Overview not showing
Go to 🏷 Categories and enter a ₹ amount in the budget field next to any category. The section is always visible and shows a prompt if no budgets are set.
⚠ New category not saving budget to cloud
Run this SQL in Supabase: alter table categories add column if not exists budget numeric default 0;
⚠ Charts not rendering
Chart.js is loaded from CDN — check your internet connection. Charts auto-destroy before re-drawing to prevent memory issues.
⚠ Can't install as PWA on Android
Make sure manifest.json is in the repo and linked in <head>. GitHub Pages uses HTTPS which is required for PWA install.
